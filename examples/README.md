# Examples

This directory contains two reference examples generated by `claude-project-bootstrap`.
They demonstrate what the tool produces after a completed 5-step bootstrap session.

## lagrangia/

**Project:** Lagrangia MecAI — a CAD code generation system at Dessia Technologies.

**Type:** Existing codebase — monorepo-to-multirepo migration.

**Complexity:** High
- 5 phases
- 7 agents (4 Sonnet, 3 Haiku)
- 16 stories across phases

**Highlights:**
- Demonstrates how the methodology handles complex multi-repo migration
- Shows strict constraint encoding (no API breaking changes, CI always green)
- STORY-001 (Phase 1) and STORY-003 (Phase 2) are fully fleshed with real verification commands
- Illustrates specialised domain agents (architect-multiagents, dev-memory, dev-tools)

This is the **reference case**. Study it to understand how the methodology scales to
industrial-grade projects with non-trivial constraints.

## minimal/

**Project:** mylib — a simple greenfield Python utility library.

**Type:** Greenfield — no existing code.

**Complexity:** Low
- 2 phases
- 3 agents (1 Sonnet, 2 Haiku)
- 4 stories

**Highlights:**
- Demonstrates the minimum viable scaffold for a new project
- Standard Python `src/` layout with pytest
- STORY-001 (foundation) is fully fleshed

This is the **baseline case**. Start here if you are new to the methodology.

## Relationship to Templates

All files in these examples were generated from the templates in `../templates/`.
They show the scaffolder agent's output with every template variable substituted —
no unfilled placeholders remain. These are production-ready scaffold outputs.

## How to Read an Example

1. Start with `project_context.md` — understand the project and its constraints
2. Read `roadmap.md` — understand the phase structure and dependencies
3. Read `.claude/CLAUDE.md` — understand the project constitution and agent ownership
4. Read `.claude/stories/STORY-001.md` — understand what a complete story looks like
5. Read `.claude/agents/` — understand how agent files are structured
